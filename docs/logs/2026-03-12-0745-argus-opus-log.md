## 07:55 — Markdown heading investigation (Theseus assignment)

**Finding: Not a bug.** Traced the full pipeline:
1. `import.ts:112` — `fs.readFileSync(claudeMdPath, 'utf-8')` — raw read, no processing
2. Stored as `sourceMetadata.claudeMd` → `JSON.stringify()` into `source_metadata` column
3. `buildKitBriefing:196` — retrieves via `JSON.parse`, wraps with prefix, no stripping
4. `buildSystemPrompt:218` — joins parts with `\n\n`, no transformation

The `#` heading syntax is preserved byte-for-byte through the entire pipeline. Secundus's omission when quoting "first 25 words" is the same **lexical drift** documented in quiz Q7 — the model reconstructs from understanding rather than reproducing exact surface form. This is inherent to LLM processing, not a code bug.

**However** — worth noting that Secundus reported markdown was otherwise present in their context (headers, backticks, tables). The delta is narrow (first-line heading only) and is model behavior, not import behavior.
